rndm.nu

True Random Number Web Service

API

rndm.nu/{language/}{mode}[mode options]{.format}

rndm.nu provides a simple URL based API to retrieve true random numbers in all sorts of formats. The API is easy to use for both humans and programs alike. Above is the basic format which every API request to rndm.nu must use. Parameter names are are in bold. Optional parameters are surrounded by {curly brackets}. Mandatory parameters are surrounded by [square brackets]. Below the parameters and how to use them are described in detail with examples for each use case.

Language

{language/} where language must be either 'en', 'es', 'ja' or 'ch'

Represents the ISO 639-1 language code of the language you prefer HTML responses and error messages to use. If not provided the the language will be chosen from the Accept-Language header in the HTTP request. If no preference is provided in Accept-Language the language will default to 'en'.

Mode

[mode] where mode must be either 'standard', 'binary', 'shuffle' or 'lottery'

The mode parameter determines what kind of random numbers you want in your response. If mode is not provided it will default to 'standard'. Each mode and it's corresponding mode options are described further down.

Format

{.format} where format is either 'html', 'txt', 'json' or 'bin'

Represents the format you want the numbers returned in. Defaults to html. If format is not provided in the URL it can be set by specifying the corresponding MIME type in the HTTP request 'accepts' headers.

html (MIME type: 'html') : Web-browser view-able HTML result.

txt (MIME type: 'txt'): human readable text response, also easily parse-able programmatically.

json (MIME type: 'json'): returns numbers in JSON format.

bin (MIME type: 'octet-stream'): returns numbers in binary format. Each number is a 64-bit integer in big-endian format.

Modes

The rndm.nu API offers four different modes with which to retrive random numbers of multiple kinds. They are described as follows:

Standard mode

/[range]{xmultiplier}{.format}

Standard mode returns well formatted random numbers with many formatting options.

eg:

rndm.nu/10 (a random number between 1 and 10 in html format)

rndm.nu/5x100.txt (100 random numbers between 1 and 5 in text format)

rndm.nu/3-4.bin (a random number between 3 and 5 in binary format)

rndm.nu/500-600x1000.json (1000 random numbers between 500 and 600 in json format)

Range

[range] where range > 1 or [range1-range2] where range2 > range1 and range1 >= 0

Represents the integer range a random number will be picked from.

eg:

5 (number between 1 and 5)

6-8 (number between 6 and 8)

Multiplier

{xmultiplier} where multiplier > 0

Represents how many individual random numbers you want returned.

eg:

x10 (10 random numbers)

Binary mode

/binary[amount]

Returns a stream of pure unbiased random bytes. Binary mode will only return a bin format response. html, txt and json formats are not supported in Binary mode.

eg:

rndm.nu/binary50 (returns 50 random bytes)

rndm.nu/binary1024 (returns 1kb of random bytes)

Amount

[amount] where amount > 0

Specifies the number of bytes you wish to be returned.

Shuffle mode

/shuffle[length]

Shuffles a consecutive set of integers.

eg:

rndm.nu/shuffle100 (a shuffled list of numbers from 1 to 100 in html format)

rndm.nu/shuffle1024.json (a shuffled list of numbers from 1 to 1024 in json format)

Length

[length] where length > 1

Specifies the the length of the shuffled number list.

Lottery mode

/lottery[range][-amount]{.format}

Randomly picks a specified amount of distinct integers from a range of integers.

eg:

rndm.nu/lottory48x6 (a list of 6 randomly picked integers between 1 and 48 in html format)

rndm.nu/lottory36x7.txt (a list of 7 randomly picked integers between 1 and 36 in txt format)

Range

[range] where range > 1

Specifies the range of numbers to pick from.

Amount

[-amount] where amount > 0 and amount <= range

Specifies the number of integers to pick and return

Errors

In the event of an error, such as badly formatted parameters, the API will return an error object. An Error response will always be associated with a 400 HTTP status code.

The error object will have the following fields:

Error code

An integer representing the error type id.

Language Code

The ISO 639-1 language code of the error name and description.

Friendly Name

A human readable name of the error.

Description

A human readable detailed description of the error.

The format of the error message will depend on what format is specified in the parameters. If none are specified the 'accepts' header in the HTTP request will be used to choose the error format. If neither the URL parameters or the 'accepts' header specify a preference for a format, then html will be used. The format of the error object can always be known by checking the MIME type defined in the Content-Type header in the HTTP response.

JSON

A JSON format error object will be in the following format:

{
    errorCode: [Error Code (number)],
    languageCode: [Description (string)]],
    friendlyName: [Friendly Name (string),
    description: [Description (string)]
}

The MIME type is: 'json'

eg:

rndm.nu/parseError.json

rndm.nu/ja/parseError.json (Returns a JSON parsing error object with Japanese friendlyname and description)

BIN

A BIN format error object will be in the following format:

byte 0-4: 32-bit unsigned big-endian integer representing the Error Code

byte 6-7: 2 ASCII characters representing the Language Code

byte 8-*: Null terminated UTF-8 string representing the Friendly Name followed immediately by a second null terminated UTF-8 string representing the Description

The MIME type is: 'octet-stream'

eg:

rndm.nu/parseError.bin

TXT

A TXT format error object will be in the following format:

Error Code: [Error Code],
Language Code: [Language Code],
Friendly Name: "[Friendly Name]",
Description: "[Description]"

The MIME type is: 'txt'

The field names on the left will appear in the language represented by the language code.

eg:

rndm.nu/parseError.txt

rndm.nu/es/parseError.txt

(Returns a TXT parsing error object with Spanish friendly name and description)

HTML

HTML formats are aimed to be human readable and can be viewed in a standard web browser. The MIME type is 'html'.

eg:

rndm.nu/parseError.html

Quota

rndm.nu uses an IP based quota system to help prevent exhausting resources. More details on the quota system can be read at rndm.nu/qouta.

Details about the current status of your quota can be read from any HTTP response headers which will always have the following headers:

x-number-quota-remaining

The number of individual numbers remaining in your quota.

x-request-quota-remaining

The number of requests remaining in your quota.

x-raw-binary-quota-remaining

The number of raw binary bytes remaining in your quota.

x-quota-reset-date

The time and date that your quota will reset next in UCT format.

For performance reasons the quota details described in these headers are not guaranteed to be 100% up to date. For guaranteed up to date quota details go to:

rndm.nu/qouta

or

rndm.nu/qouta.json which will return a JSON object with the following fields:

numberQuotaRemaining (equivalent of x-number-quota-remaining)

requestQuotaRemaining (equivalent of x-request-quota-remaining)

rawBinaryQuotaRemaining (equivalent of x-raw-binary-quota-remaining)

quotaResetDate (equivalent of x-quota-reset-dat)
